# Security Policy

## Reporting a vulnerability

If you believe you've found a security issue in any Sleepy Productions project, please report it privately rather than opening a public issue.

- Use GitHub's **private vulnerability reporting** (the "Report a vulnerability" button under a repo's Security tab) where available, or
- Open a minimal issue asking for a private contact channel — do **not** include exploit details in the public issue.

Please include enough detail to reproduce the issue. We'll acknowledge reports as quickly as we reasonably can and keep you updated on the fix.

## Scope

These projects are local-first and privacy-respecting by design. Reports about data exfiltration, credential handling, or unexpected network calls are especially welcome.
